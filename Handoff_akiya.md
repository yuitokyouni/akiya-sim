# HANDOFF — 空き家政策シミュレータ（都知事杯オープンデータ・ハッカソン2026）

作成: 2026-07-11 / 出所: claude.ai 企画セッション / model=v0.4-stock-renewal  
更新: 2026-07-21 — **主人公を行政→都民へ交代**（プロダクト作り直しではなく問いの交代）

## 0. このプロジェクトは何か

都知事杯オープンデータ・ハッカソン2026 への応募作品。

**正面の問い（都民）**: 「うちの街は10年後どうなってる？」  
**裏側の装置（行政）**: 空き家ABMによる政策シミュレーション（税・解体補助の同一シード比較）

エンジン・地図・データは捨てない。変えるのは**主語**と**入口**だけ。
審査のソーシャルインパクト／サービスデザインは都民主語。過去受賞も同型。
クラッソーネの轍（プロダクト全面作り直し）は踏まない——**主人公の交代**で足りる。

- エントリー文（確定・50字）:
  「空き家数の増減について補助金や税金といった政策による効果を地域ごとに分析可能なシミュレーションモデル」
- 対外ピッチの前面: 個人の不安 → 自分の街の事実と10年レンジ → 仕組み → 変えられること
  （YORUMICHI同型。静的計算機との差は「時間」と「仕組みの説明」）
- provenance / 監査可能性は最後に1枚。前面に出さない。

## 0.1 都民向け4枚構成（トップは検索窓一つ）

町丁目または駅名を入れると順に出る:

| 枚 | 内容 | データの性格 |
|----|------|----------------|
| **いま** | 空き家率・高齢化・取引の動き | **実データ＝事実**（予測ではない） |
| **このまま10年** | シミュレーション将来レンジ | **幅**で見せる。断定禁止 |
| **なぜ** | 相続→放置が合理的になる構造 | セル・インスペクタ流用 |
| **変えられること** | 区の制度＋「もし区がこうしたら」 | 現行政策シミュは**ここに格納** |

捨てる画面はゼロ。行政向けUIは4枚目の裏の演出装置。

## 3. モデル仕様（v0.3）

### 3.1 状態
- セル=建物。状態: S_OCC(居住) / S_RENT(賃貸中) / S_SALE(売出) / S_LIST(募集) / S_NEG(放置) / S_DEMO(更地・域外マスク兼用)
- 所有者属性: oAge(年齢), oRemote(遠隔居住), oCo(共有名義人数1-3), attach(愛着・保留価値), negYears(放置年数)
- 1ステップ=1年。ペア実行はワールドA(政策ゼロ)とB(つまみ適用)を同一seed(mulberry32)で駆動。

### 3.2 地域（ZONES定数、実データ差替え対象）
西多摩 / 多摩中部 / 多摩東部 / 区部西 / 都心 / 城東。
各地域: price(価格水準) / inc(所得指数) / mig(年純流入率) / ageS(所有者年齢シフト)。
効き方: price→立地係数loc、inc・mig→需要係数zDem(成約見込みの乗数)、mig→転出率、ageS→初期年齢分布。

### 3.3 毎年の遷移（v0.4）
1. **更地再利用** (`rebuild*`): 需要×価格が高いほど `S_DEMO→S_OCC/S_RENT`（ストック更新）。
2. 加齢 → 死亡 → **相続分岐** (`inheritDisposition`): 遠隔・共有ほど `S_NEG`、それ以外は `S_SALE`/`S_LIST`。
3. 転出分岐（売出/募集/放置）・退去（多くは再募集）。投資的募集の薄いフロー。
4. 空き家所有者の意思決定。決定確率 pDecide=(1/oCo)×(remote?0.55:1)×(1+0.25*tax)。
   効用: uNeg / uSell / uRent / uDemo（規範・家計余裕・区切り解体・補助実効を含む）。
   m = 成約見込み（立地差を抑えて住調の平坦さに整合）。
5. マッチング: S_SALE/S_LIST が確率mで成約。品質劣化・近傍再計算。

### 3.4 検証済みの創発（seed=42, 住調2023 初期化, N03 zone[]）

- **初期条件 (t=0)**: 6地域空き家率を住調2023にモーメントマッチ。`docs/validation_jyutaku2023.md`。
- **短期 (t=10)**: 全体空き家率 ~11–12%（`harness_10y.js`）。
- **ウォークフォワード**: 2008→2013 / 2013→2018 / 2018→2023、mean MAE ≈ 3.2pp（`harness_walkforward.js`）。
- **長期 (t=60)**: 空き家 ~10%、ベース更地 ~0%（再利用で吸収）、介入更地 ~16%。
- zone[] は N03 ラスタライズ（`docs/n03_zone_grid.md`）。

## 4. キャリブレーション履歴（同じ轍を踏まないこと）

- v0.1: 需要を固定到着数にした → 死亡・転出フローに追いつかず空き家率93%に発散、政策効果ゼロ。
  修正: 需要を物件ごとの成約見込みmに変更。
- v0.2初期: 「賃貸」が常勝し放置が出ない → 効用に成約見込みmを乗算、行動に意思決定コストeffを導入。
- 次の問題: 拘束条件がペイオフでなく意思決定タイミングになり税が無効化 →
  attach(放置の積極的理由)と税の決定強制(pDecide乗数)を導入。← この診断自体がプレゼン素材。
- 高摩擦所有者が永久放置の吸収状態化 → 摩擦疲労項(negYearsで減衰)を導入。
- 西多摩の発散(66%) → 純流入弾力とmoveOut増幅を緩和して53%まで調整。まだ実勢(奥多摩で3割程度)より高い。
- v0.3.1: ZONES.mig を住基人口移動報告(令和6年)の6地域集計に差替え(`data/zones_mig.csv`)。
  合成値は西多摩のみ純流出だったが実績は全域純流入。勾配(西>東の空き家率順序)は seed=42/60年で維持。
  多摩中部の空き家率は48%→28%に低下。詳細は `docs/validation_mig_r6.md`。

## 5. 既知の限界（対外説明での約束事）

- 全数値は合成格子上の挙動。**主検証は住調ウォークフォワード**（水準だけでなく5年外挿）。地物一致の予測モデルではない。
- 更地再利用は様式化（ビオトープ等の土地利用分岐は将来）。地形制約なし。マップは「様式化」と即答。
- stylized-fact一致はメカニズムを識別しない(本人のAtlas論点)。ハッカソンでは割り切り、
  WP化する場合のみ介入データ(特定区の施策前後)での検証を検討。

## 6. データ計画（実データ差替え）

Phase 0(go/no-go, 未実施):
1. 空き家所有者実態調査(令和6年, e-Stat)に東京都/大都市圏の層別集計があるか ← 最優先。無ければ全国分布で代用し主張を弱める。
2. PLATEAU 23区の建築年属性の充足率。
3. 対象区の空き家実態調査PDF(世田谷・大田等)の有無。

パラメータ↔データ対応:
- 所有者分布(oAge, oRemote, oCo, 相続割合, attach規模感) ← 空き家所有者実態調査
- 検証ターゲット(区別空き家率・種類別) ← 住宅・土地統計調査2023(区市町村)。「その他空き家」=放置の代理変数
- ZONES.mig ← 住民基本台帳人口移動報告(区×月次) / ZONES.inc ← 市町村税課税状況等の調
- 価格場 ← 路線価(既存OCRパイプライン, PaddleOCR方針) + 不動産情報ライブラリAPI(成約, 町丁目×四半期)
- ストック ← PLATEAU(建物単位) or 土地利用現況GIS。都GISは平面直角第9系→WGS84変換が必要
- **空間マスク zone[]** ← N03 行政区域ラスタライズ（`docs/n03_zone_grid.md`, v0.3.2）。手描きシルエット廃止。
- 人口動態 ← 国勢調査小地域(町丁目, 高齢単身世帯→死亡→相続の駆動系) + 住基月次(都データ, 主軸に見せる)

## 7. エンジニアリング規約（維持すること）

- 単一HTML・依存最小を維持。パラメータはPブロックとZONES定数に集約(キャリブレーション対象を一箇所に)。
- provenance行(seed/政策/model版数)は常時表示。版数はモデル変更ごとに上げる。
- **ヘッドレス検証ループを必ず回す**: <script>を抽出 → node --check → **10年ハーネス（主）** `scripts/harness_10y.js` → 60年ハーネス（副）で
  (1)住調2023 初期一致 (2)10年レンジ (3)長期非発散 (4)政策の用量反応 を数値確認してからUIを触る。
  ブラウザで見た目だけ確認して出荷しない。
- 数値主張はすべて seed と年数を添えて記録する。
