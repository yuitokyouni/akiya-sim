# ウォークフォワード検証（大学受験過去問理論）

## 一言で

**n年までのデータで初期化 → n+5年を予測 → 答え合わせ →（答えを見ずに）次の過去問へ。**

いままで t=0 を住調2023に合わせるだけで、動学の外挿力は測っていなかった。
本手順はその穴を塞ぐ。

## 手続き

| ステップ | 内容 | カンニング禁止事項 |
|----------|------|-------------------|
| 1 | 住調 wave *t* の6地域空き家率で `ZONES_VAC.rates` をセット | 先の波の率を使わない |
| 2 | 現行 `engine.js` の **P 固定**で 5年 `step` | 答えを見て P をいじらない |
| 3 | wave *t+5* の地域別率と比較（MAE / RMSE / 符号付き誤差） | — |
| 4 | ずれを記録。補正するなら **学習区間だけ**（例: 〜2018）で再フィットし、最終ホールドアウト（2018→2023）は最後に一度だけ見る | 2023を見ながら2018用Pを最適化しない |

## ホップ定義

1. **2008→2013**
2. **2013→2018**
3. **2018→2023**（いちばんきつい最終試験）

空き家の種類別比率（neg/sale/list）は当面 **2023分布を全波に代理適用**（種類表の時系列整備は次タスク）。

## ゲート

| 種別 | 閾値 | 意味 |
|------|------|------|
| FAIL | t=0 初期化の全体率がターゲットから **>1.5pp** | 入力バグ |
| WARN | 地域MAE **>4pp** | 過去問のズレ＝補正候補 |
| FAIL | 地域MAE **>8pp** | オーダー破綻 |

## 現状の診断（現行P・seed=42）

直近ラン（`docs/validation_walkforward_latest.json`）:

| ホップ | 全体誤差 | 地域MAE |
|--------|----------|---------|
| 2008→2013 | **+8.5pp** | 6.1pp |
| 2013→2018 | **+9.8pp** | 7.0pp |
| 2018→2023 | **+5.8pp** | 4.7pp |

- 3ホップとも **空き家率を過大予測**（5年でだいたい +6〜10pp）
- mean MAE ≈ **5.9pp**（WARN帯・壊滅的な8pp超ではない）
- 解釈: t=0フィットだけでは動学が「空き家が増えすぎる」。次は学習区間（2008→2013と2013→2018）だけで `holdCost` / `match` 等を補正し、**2018→2023をホールドアウト**する

## やらないこと

- 地物一致の街区予測（HANDOFF §5）
- 更地率の住調突合（住調に更地ストックはない）
- 答え（次波）を見ながらのパラメータ一発合わせ
