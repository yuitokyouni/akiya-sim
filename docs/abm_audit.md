## 状態定義（用語の整理）

| コード | 名称 | 意味 | 住調との対応 |
|--------|------|------|--------------|
| `S_OCC` | 居住 | 所有者または入居者がいる | 居住世帯あり |
| `S_RENT` | 賃貸中 | 入居者あり（賃貸） | 居住世帯あり |
| `S_SALE` | 売出中 | 空き家・売却活動中 | 空き家（売却中） |
| `S_LIST` | 募集中 | 空き家・賃貸募集 | 空き家（賃貸募集） |
| `S_NEG` | **放置** | 空き家のうち積極的処分なし | 住調「その他空き家」の代理 |
| `S_DEMO` | **更地** | 建物解体済み・土地のみ | 住調の空き家からは**除外** |

### 空き家 vs 放置 vs 更地

- **空き家率**（モデル集計 `hist.vac`）= `(S_SALE + S_LIST + S_NEG) / 全セル`
  - 売出・募集中も含む「居住世帯のない建物」
- **放置率**（`hist.neg`）= `S_NEG / 全セル` — 空き家の**部分集合**
- **更地率**（`hist.demo`）= `S_DEMO / 全セル` — 空き家**ではない**（建物ストックから脱落）

## Phase 1: 非経済経路（簡易モデル）

家計余裕の代理: `ownerWealth(w,i) = loc × q × ZONES[zone].inc`

| パラメータ | 意味 | 効く効用 |
|------------|------|----------|
| `normK` | 社会的規範・周囲の目（余裕層ほど放置しにくい） | `uNeg` を悪化、`pDecide` 微増 |
| `poorBar` | 家計余裕不足（前払い負担） | `uRent`, `uDemo` に `cashStress`。**`uSell` には非適用** |
| `closureK` | 区切り解体（遠隔・共有・長期放置） | `uDemo` に `closureRelief` |
| — | 老朽化（15年超放置） | `structuralClosure` + 解体の `cashStress` 緩和 |

**解釈（ユーザー仮説の代理）**

- 余裕層: 周囲の目・社会的規範で長期 `S_NEG` を避け、売却・賃貸へ
- 余裕不足層: 改修・解体の前払いが重く `cashStress` で行動しにくい → 放置に留まりやすい
- 長期放置・遠隔・共有名義: 非金銭的「区切り」としてベースでも少量の `S_DEMO` が発生

## 政策が効く経路（監査結果）

```
pol.tax ──┬──► holdTax 増（保有コスト）
          └──► pDecide 増（意思決定頻度）──► 売却/賃貸/解体/放置の再選択

pol.sub ──────► uDemo のみ（解体効用のコスト項）──► demo 選択時のみ
```

| 政策 | 効く変数 | 効かない経路 | 検証 |
|------|----------|--------------|------|
| 税 2% | `holdTax`, `pDecide` | `uDemo` 直接 | `harness_audit.js` |
| 補助 150万 | `uDemo` の `-pol.sub` | `pDecide`, `holdTax` | 同上 |
| 税+補助 | 両方 | — | 超加法性は近似 |

**旧バグ（修正済）**: 解体 0 件なのに補助スライダーが vac を変えたように見えた → 実際は tax のみが効いていた。補助単独では vac/demo 不変だった。

## 既知の設計上の限界

1. **更地の再利用なし** — `S_DEMO` は終端状態。新築・建替え・ビオトープ化フロー未実装。
2. **マスクと更地の型共有** — 域外 `zone[i]<0` も `S_DEMO` だが集計から除外。
3. **`step` 内の `S_DEMO` スキップ** — 更地セルは以降一切更新されない（意図的）。
4. **同一シード** — `makeWorld(seed)` は policy 非依存。A/B 比較は妥当。
5. **パラメータ** — `demoBase`, `attach`, `match` 等は様式化。住調は t=0 水準のみ。
